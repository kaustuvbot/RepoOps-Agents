## Purpose

This skill tells Codex:
- what stack the repo uses
- where core code lives
- which commands are authoritative
- which architectural rules must be respected

## Rules

- Prefer the documented project structure over inventing a new one.
- If the architecture file is incomplete, say what is missing.
- If a task conflicts with documented repo boundaries, stop and call it out.
